# v0.1 成果汇报

> 公开展示边界：本文件只保留聚合后的工程结果与限制说明。真实视频运行目录、全量标注、seed JSONL、切分文件和实验日志包含第三方平台内容或内部审核材料，均不在本次求职展示快照中公开。因此，以下数字用于说明本地流水线的完成状态，不构成可由公开仓库独立复现的模型结论。

当前 v0.1 的结果可以概括为：

**数据流水线合格，baseline 跑通；但模型能力还很弱，尤其不能视为 truthfulness 判断模型。**

## 1. 数据与执行状态

v0.1 共生成 `424` 条 B站 seed 样本，其中 `414` 条进入主训练/评测任务，`10` 条因为 `claim_text` 超过 300 字、仍偏向整段转写，被排除出主任务。

预检查结果是合格的：

- `hard_error_count = 0`
- 初始重复文本冲突 `9` 组，已通过 `17` 条 atomic split repair 修复
- 修复后重复冲突 `0`
- `raw_context` 标签泄漏 `0`
- 必填字段缺失 `0`
- 重复 `source_id + claim_id` `0`

这说明 v0.1 的主要目标之一已经达成：数据转换、schema 校验、泄漏检查、切分、baseline 评测链路都跑通了。

切分情况：

| split | records |
|---|---:|
| train | 294 |
| dev | 57 |
| test | 63 |

另外也生成了 5 折 GroupKFold，用来降低单次视频级切分的偶然性。

## 2. 最好的任务：include_decision

这是当前表现最好的任务，目标是判断 claim 是否应进入训练池。

holdout test：

- accuracy `0.8413`
- macro F1 `0.6420`
- weighted F1 `0.8497`

5 折 GroupKFold：

- macro F1 mean `0.6714`
- accuracy mean `0.8444`

但要注意：`included` 样本明显更多，test 上 `included` 的 F1 是 `0.91`，而 `excluded` 的 F1 只有 `0.38`。也就是说模型已经能大致学到“多数可入库 claim”的模式，但对需要排除的边界样本识别仍不稳定。

结论：`include_decision` 可以作为 v0.1 可用的弱 baseline，但还不能自动替代人工过滤。

## 3. claim_type：结果偏弱

holdout test：

- accuracy `0.6508`
- macro F1 `0.2031`
- weighted F1 `0.5562`

5 折 GroupKFold：

- macro F1 mean `0.1996`
- accuracy mean `0.5440`

问题很明显：模型大量预测 `numeric_statistical_claim`。test 上：

- `numeric_statistical_claim` F1 `0.78`
- `factual_descriptive_claim` F1 `0.24`
- `allegation_or_reported_claim` F1 `0`
- `opinion_normative_claim` F1 `0`
- `product_experience_claim` 在 test 中 support 为 `0`

`advertising_or_sponsor_claim` 全库只有 `3` 条，已经被标为 observe-only，不计入主指标。

结论：claim_type 目前只能说明评测链路可用，分类器本身还不可靠。

## 4. checkability：基本是多数类模型

holdout test：

- accuracy `0.6190`
- macro F1 `0.1529`
- weighted F1 `0.4734`

5 折 GroupKFold：

- macro F1 mean `0.1541`
- accuracy mean `0.6414`

test 上模型几乎只会预测 `indirectly_checkable`：

- `indirectly_checkable` F1 `0.76`
- `context_dependent` F1 `0`
- `directly_checkable` F1 `0`
- `not_checkable` F1 `0`

结论：checkability 任务目前不具备可用模型能力，需要更多均衡样本和更清晰的判定边界。

## 5. truthfulness_status_calibration：不能当真实性模型

这个任务在 v0.1 里只是 calibration，不是正式 truthfulness 判断。

holdout test：

- accuracy `0.7222`
- macro F1 `0.2097`
- weighted F1 `0.6057`

5 折 GroupKFold：

- macro F1 mean `0.2242`
- accuracy mean `0.8157`

表面 accuracy 不低，但原因是 `gold_supports` 占绝对多数。test 上：

- `gold_supports` F1 `0.84`
- `gold_misleading` F1 `0`
- `gold_missing_context` F1 `0`
- `gold_partially_supports` F1 `0`

模型几乎把所有样本都判成 `gold_supports`。

结论：这个结果不能说明模型会判断真假，只说明当前数据分布下，多数类策略也能拿到较高 accuracy。v0.1 不能宣传为事实核查模型。

## 6. noise_patterns：只有少数模式有信号

holdout test：

- macro F1 `0.1026`
- micro F1 `0.3128`

5 折 GroupKFold：

- macro F1 mean `0.0920`
- micro F1 mean `0.2387`

多数噪声标签 F1 为 `0`，但两个模式有明显信号：

- `time_mismatch`: test F1 `0.67`
- `unit_or_scale_error`: test F1 `0.67`

这说明字符级 TF-IDF 对时间、单位、规模类表述能抓到一些表面模式，但对 `missing_context`、`source_laundering`、`opinion_as_fact`、`partial_truth` 等语义型噪声还不行。

## 总体判断

v0.1 当前结果是一个合格的“数据种子池 + schema + baseline smoke”结果：

- 可以证明数据产物可生成、可校验、可切分、可重复评测。
- 可以保留 `include_decision` 作为弱 baseline。
- 不能证明模型具备 truthfulness 判断能力。
- `claim_type`、`checkability`、`truthfulness_status_calibration`、`noise_patterns` 都主要暴露了样本不均衡和视频级强相关问题。
