# Stage 5 V02 synthetic contracts

This directory documents the public synthetic boundary for G0B. It intentionally contains no persisted business payload fixture, real video, audio, transcript, Gemini answer, credential, private path or model output.

Tests generate every fixture deterministically inside pytest `tmp_path`, validate it, and discard it. G0B therefore exercises contracts without creating a second public data authority or reading the sealed real run.

## Generation rules

Synthetic fixtures use:

- fixed canonical IDs with valid typed ULID shapes;
- fixed UTC `Z` timestamps and deterministic ordering;
- repository-relative POSIX paths only;
- canonical JSON plus one LF, or canonical JSONL with one LF per record;
- SHA-256 over the exact generated bytes;
- embedded object hashes recomputed only after all semantic fields are fixed;
- explicit Workflow/DAG/Schema/Profile/Prompt versions;
- `public_synthetic` privacy and public access only for invented content.

No fixture copies bytes, names, URLs, excerpts or claims from the real Stage 5 input.

## Covered flows

The generated cases cover:

1. S01 business payload validation and entity/Artifact locator closure.
2. A no-depth route that produces `screening.sync_record` and stops at an unauthorized S03 route.
3. A depth-wait route that publishes `source_depth.prompt`, closes S02-A as `WAITING_FOR_HUMAN`, and has no running Agent, poller, timer or collector afterward.
4. A synthetic G2 capture in which one exact `.json`, `.md` or `.txt` file is stat-read-hash-restat, remains read-only, and is wrapped in a create-new canonical `source_depth.result` envelope.
5. A separate G3 import Session that treats every returned item as a lead, maps claim IDs, deduplicates sources and preserves unresolved conflicts.
6. Exact HANDOFF/checkpoint recovery with zero writes.
7. Per-Session model telemetry described in `model_telemetry/README.md`.

The synthetic depth case does not call Gemini. Its invented return file only verifies the same capture and import structure that a later user-provided Gemini Web result must satisfy.

## Negative cases

Tests fail closed on Schema/version drift, orphan entity locators, unknown upstream Artifacts, second source media, non-canonical or changed bytes, unauthorized paths, glob/`latest`, reparse points, oversized or unstable manual input, missing G2/G3 gates, direct Gold promotion, Token estimation, telemetry chain breaks and writes after close.

Passing these cases authorizes only the G0B implementation/synthetic boundary. It does not authorize network access, model download, real-media reads, Gemini automation, a real S01/S02 run or automatic DAG traversal.
